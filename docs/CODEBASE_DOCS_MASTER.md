# Neurotoxic Architecture (Code-Synced)

_Last verified against code: 2026-04-13._

## Runtime stack

- React `19.2.5`, React DOM `19.2.5`
- Vite `8.0.9`
- Tailwind CSS `4.2.2`
- Framer Motion `12.38.0`
- Tone.js `15.5.6`
- Node engine: `>=22.13.0`

## App composition

- Entry point: `src/main.jsx`.
- Root app: `src/App.jsx`.
- Global state: `src/context/GameState.jsx` + reducer map in `src/context/gameReducer.js`.
- Scene/phase constants: `src/context/gameConstants.js`.
- Main scenes live under `src/scenes/*`.

## Core domains

- **State and reducer domains:** player, band, social, gig, events, minigames, system, quests, clinic, trade.
- **Economy:** `src/utils/economyEngine.js` (`MODIFIER_COSTS` is canonical).
- **Events:** `src/data/events/*` aggregated in `src/data/events/index.js`.
- **Audio:** `src/utils/audio/*` and audio manager/engine wrappers.
- **Travel and map:** `src/hooks/useTravelLogic.js`, `src/hooks/useArrivalLogic.js`, `src/utils/mapUtils.js`.

## State invariants currently enforced

- Player money is clamped by helpers (`clampPlayerMoney`).
- Band harmony is bounded using state utility helpers.
- `START_GIG` flow resets gig modifiers to defaults.
- `useArrivalLogic` owns arrival routing behavior.

# Coding Standards (Code-Synced)

_Last verified: 2026-04-13._

## Tooling + package rules

- Use **pnpm only**.
- Keep pinned core versions (React, Vite, Tailwind v4, Framer Motion, Tone.js).
- Do not introduce Howler.js.

## Frontend and styling rules

- Tailwind v4 syntax only (`@import 'tailwindcss'` in CSS).
- Never hardcode app theme colors in UI logic; use CSS tokens (`var(--color-*)`).
- For Pixi color tokens, use `getPixiColorFromToken('--token')` (without `--color-` prefix).

## React rules

- React 19 pattern: pass `ref` as a normal prop; avoid new `forwardRef` usage.
- Include `t` in hook/callback dependency arrays when used in the closure.
- Keep UI text in i18n (`t(...)` / `<Trans>`), namespaced keys.

## State management rules

- Add state actions in three places together:
  1. `src/context/actionTypes.js`
  2. reducer handling (`src/context/gameReducer.js` or domain reducer)
  3. `src/context/actionCreators.js`
- Use action creators, not direct reducer invocation.
- Preserve state bounds (money >= 0, harmony 1..100).

## Testing rules

- Legacy logic tests: `pnpm run test`
- UI + migrated suites: `pnpm run test:ui`
- Full gate before PR: `pnpm run test:all`
- Do not mix node:test and Vitest styles within one test file.

## Audio + minigame rules

- Use `audioEngine.getGigTimeMs()` as canonical gig clock source.
- Do not import PIXI inside minigame logic hooks.

# Neurotoxic Game Threat Model (Current)

_Last reviewed: 2026-04-13._

## System context

- Client-rendered React/Vite game with local persistence and static assets.
- No server-side gameplay authority in this repository.

## Primary threat classes

1. **Client-state tampering** (localStorage manipulation / cheating).
2. **Supply-chain compromise** (malicious dependency updates).
3. **Static asset integrity risk** (tampered deployed bundles/assets).
4. **Future XSS regressions** if unsafe HTML sinks are introduced.
5. **Client-side DoS/perf abuse** through pathological interactions.

## Existing defensive posture

- Centralized state management and validation-oriented utility patterns.
- Limited trusted rendering surface via React default escaping.
- Strong version pinning for major runtime dependencies.

## Recommended ongoing controls

- Continue strict dependency pinning and lockfile review.
- Keep ESLint/review guardrails for DOM sink usage.
- Preserve robust load-time state validation for persisted saves.
- Run full test gates before releases/PR merges.

# Security Best Practices Report (Current)

_Last reviewed: 2026-04-13._

## Operational baseline

- Use pinned dependencies and pnpm lockfile discipline.
- Keep Node runtime at `>=22.13.0` as declared.
- Run full suite before merge: `pnpm run test:all`.

## Frontend safety posture

- Treat all persisted client state as untrusted input.
- Avoid dynamic `innerHTML` or equivalent unsafe sinks.
- Keep user-facing copy in i18n resources to reduce ad-hoc string injection paths.

## High-value checklist for contributors

1. Validate/guard loaded localStorage payloads.
2. Maintain bounded numeric state (money/harmony/etc.).
3. Avoid introducing direct Tone.js or PIXI usage in restricted layers.
4. Keep EN/DE locale files synchronized with UI text additions.
5. Re-check threat assumptions when adding external APIs or backend components.
