# src/ui/overworld - Agent Instructions

- Overworld header/HUD/menu split is intentional: derive gameplay state in `src/scenes/Overworld.tsx` or an Overworld hook, then pass typed props into these view components. If a required prop is missing or invalid, fix the caller/type contract instead of adding local gameplay derivation.
- Event log entries must remain append-safe: append new entries with generated IDs, preserve insertion order while capping old entries, and store entry kind/payload or i18n keys, never pre-translated strings. Translate at render with `defaultValue` fallbacks for missing keys.
- `OverworldMenu` distinguishes scene-change actions (`openClinic`, `openAssets`) from modal-open actions (`openHQ`, `openQuests`, `openStash`, `openPirateRadio`, `openMerchPress`, `openBloodBank`, `openDarkWebLeak`). Scene-change items dispatch `changeScene(GAME_PHASES.X)` from `src/scenes/Overworld.tsx`; modal-open items toggle local boolean state in `useOverworldModals`. When adding a new MANAGEMENT/HUSTLES item, mirror the pattern used by its category siblings and add the action key to the `MenuAction` union AND the `actions` map so the `satisfies Record<MenuAction, () => void>` check enforces the wiring.
- INVESTMENTS (`openAssets`) is a scene-change entry under MANAGEMENT that routes to `GAME_PHASES.ASSETS` (`AssetsScene`). Do not relocate it to HUSTLES; the hub mirrors BAND HQ as a long-term management surface.
