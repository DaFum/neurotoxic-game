# Refactor with Safety

## Workflow

1. Identify refactor boundaries and list affected modules.
2. Preserve public interfaces and behavior.
3. Run lint, tests, and build after changes.
4. Summarize risks and verification results.

## Output

- Provide a short risk report and validation status.
