## 2024-05-28 - Array .reduce() Performance in V8
**Learning:** While functional array methods generally introduce overhead, `Array.prototype.reduce()` can actually outperform manual `for` and `for...of` loops in V8 (Node 22) for simple aggregations over dense arrays (e.g., summing numeric properties across a small set of items). Replacing `.reduce()` with a `for` loop solely for object aggregation (without early returns) often results in slower execution due to V8's native optimizations for standard reduction patterns.
**Action:** Do not blindly replace `Array.prototype.reduce()` without measuring first. Only replace functional array methods if the operations are highly complex, involve intermediate allocations (like `.filter().map()`), or occur in a proven, high-frequency hot path (like 60fps render loops or audio/MIDI processing) where callback execution overhead becomes the primary bottleneck.
